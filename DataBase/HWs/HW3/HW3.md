### Задание 1 (2 балла): Создание базовой структуры

Сценарий: Вы создаете базу данных для небольшой онлайн-библиотеки. Вам нужно создать таблицу для хранения информации об авторах.

Требования:
*   Создайте таблицу Authors.
*   У каждого автора должен быть уникальный, автоматически увеличивающийся целочисленный идентификатор author_id.
*   Таблица должна хранить имя автора (full_name, до 150 символов) и год рождения (birth_year, целое число).
*   Имя автора не может быть пустым.

Что нужно сделать:
Напишите один SQL-запрос CREATE TABLE для создания таблицы Authors в соответствии с требованиями.

### Задание 2 (2 балла): Добавление связанной таблицы и ограничений целостности

Сценарий: Теперь нужно добавить таблицу для книг и связать ее с авторами.

Требования:
*   Создайте таблицу Books.
*   У каждой книги должен быть уникальный book_id (используйте SERIAL).
*   Таблица должна хранить название книги (title, до 200 символов, обязательное поле) и год публикации (publication_year).
*   Каждая книга должна быть связана с автором из таблицы Authors.
*   Бизнес-правило: Если автора удаляют из базы, вся информация о его книгах также должна быть удалена.

Что нужно сделать:
1.  Напишите CREATE TABLE для таблицы Books.
2.  Правильно определите внешний ключ (FOREIGN KEY) и укажите необходимое поведение при удалении (ON DELETE).
3.  Напишите два INSERT запроса: один для добавления автора "Александр Пушкин" (1799 г.р.), и еще один для добавления его книги "Евгений Онегин" (1833 г.р.), связав ее с только что созданным автором.
### Задание 3 (2 балла): Изменение схемы и обновление данных

Сценарий: Бизнес-требования изменились. В библиотеке теперь есть книги без авторов (например, сборники фольклора), а также нужно хранить жанр.

Требования:
*   Модифицировать таблицу Books так, чтобы поле, связывающее ее с автором, могло быть пустым (NULL).
*   Добавить в таблицу Books новое поле genre (текст, до 50 символов).
*   После изменения схемы, вставить одной командой две новые книги:
    *   "Сказки народов мира", без автора, жанр "Фольклор", 1980 г.
    *   "Пиковая дама", автор Александр Пушкин, жанр "Повесть", 1834 г.

Что нужно сделать:
1.  Напишите команду ALTER TABLE, чтобы изменить существующее ограничение внешнего ключа (подсказка: его нужно сначала удалить, а потом добавить заново с другими параметрами, или изменить саму колонку).
2.  Напишите команду ALTER TABLE, чтобы добавить новую колонку genre.
3.  Напишите один INSERT запрос для добавления обеих книг.

### Задание 4 (2 балла): Атомарная операция и возврат данных

Сценарий: Необходимо реализовать добавление нового отзыва на книгу. Процесс должен быть атомарным, и приложение сразу после добавления должно получить ID нового отзыва.

Требования:
*   Создайте таблицу Reviews со столбцами:
    *   review_id (автоинкрементный PK)
    *   book_id (внешний ключ к Books, удаление книги должно приводить к удалению отзыва)
    *   reviewer_name (текст, до 100 символов)
    *   rating (целое число, обязательно от 1 до 5)
    *   review_text (текст)
    *   created_at (timestamp с часовым поясом, по умолчанию используйте текущее время)
*   Напишите одну DML-команду, которая добавляет новый отзыв для книги "Евгений Онегин" от рецензента "Белинский" с рейтингом 5 и текстом "Энциклопедия русской жизни", и сразу же возвращает сгенерированный review_id и created_at этого отзыва.

Что нужно сделать:
1.  Написать CREATE TABLE для Reviews с CHECK constraint для рейтинга.
2.  Написать INSERT запрос с подзапросом для поиска book_id и с оператором RETURNING.
